# ttblt

A simplified implementation of [Byte Latent Transformers](https://github.com/facebookresearch/blt) as a [TorchTune](https://github.com/pytorch/torchtune) recipe, focused on fine-tuning Qwen 2.5 3B.

The original BLT repo is much more comprehensive. This project explores whether the BLT architecture can be bolted onto a pretrained model via fine-tuning on a small instruction dataset, rather than the large-scale pre-training approach used in the paper.

```
tune run full_finetune_single_device.py --config qwen2_5_3B_blt_full_single_device.yaml
```

## Architecture

Instead of traditional tokenization, the model operates on raw UTF-8 bytes:

* **Byte tokenizer** — vocabulary of 259 (256 bytes + BOS/EOS/PAD), no BPE
* **Local encoder** — 3 self-attention + 1 cross-attention layer, compresses byte sequences into patch embeddings via mean-pooling
* **Entropy-based patching** — a small (14M param) causal byte-level transformer predicts per-byte entropy; patch boundaries are placed at high-entropy positions (word/morpheme boundaries), following the BLT paper design
* **Global model** — Qwen 2.5 3B processes patch embeddings through its standard self-attention layers (no modifications to Qwen's architecture)
* **Local decoder** — 9 self-attention + 3 interleaved cross-attention layers, predicts bytes from patch representations using a shifted mask (bytes in patch p attend to patch p-1 to prevent information leak)

The full model fits in 24GB VRAM (DGX Spark GB10) with activation checkpointing.

## Training

The recipe (`full_finetune_single_device.py`) extends TorchTune's standard full fine-tune with:

* **Split learning rates** — 2e-4 for new BLT parameters, 2e-5 for pretrained Qwen (10x ratio)
* **Non-strict checkpoint loading** — new BLT parameters don't exist in the Qwen checkpoint
* **Gradient spike detection** — skips optimizer steps when grad norm exceeds 10x its EMA, protecting Adam moments
* **Pre-computed patch IDs** — entropy model runs offline to generate patch boundaries for the training set, avoiding train-time entropy model overhead
* **Periodic eval BPB** — bits-per-byte on held-out passages logged to WandB during training

Training on SlimOrca (363K examples) for 1 epoch takes ~125h on a single GB10.

## Generation

```
tune run local_generate.py --config custom_generation_config.yaml
```

Generation uses prefix patch caching — patch boundaries for the prompt are computed once, then extended incrementally for each generated byte. This ensures prefix patches remain stable (matching the causal property of the entropy model).

## Challenges and Findings

Through several rounds of training (v1-v3), a number of bugs were identified and fixed:

* **Loss mask bug** — `tokenize_messages()` was not propagating per-token masks, causing the model to train on system/user prompt tokens rather than just assistant responses
* **Decoder cross-attention leak** — bytes in patch p originally attended to patch p's embedding (which contains future byte information via pooling). Fixed with a shifted mask so bytes attend to patch p-1
* **Encoder cross-attention direction** — patches now correctly query their constituent bytes (was bytes querying bytes)
* **Batch-dependent patching** — `dynamic_patch()` shared thresholds across the batch, so patching of one example depended on others. Replaced with per-example entropy patching
* **`patch_reduce` batch contamination** — `patch_ids.max()` was computed across the entire batch rather than per-example, causing zero-padded patch embeddings to pollute attention
* **Generation config mismatches** — entropy threshold and max patch size differed between training and inference configs

## Fundamental Limitation: Data Scale

Despite fixing all identified bugs, generation quality remains poor — the model produces recognizable English fragments but not coherent text. The root cause is a fundamental data scale gap compared to the BLT paper:

| | BLT Paper | ttblt |
|---|---|---|
| Training data | 880B bytes (220B tokens) | ~150M bytes |
| Paradigm | Continued pre-training | Fine-tuning |
| Entropy model | 100M params | 14M params |

This is a **~5,000x data shortfall**. The local encoder/decoder must learn byte-to-patch and patch-to-byte mappings essentially from scratch — a task that requires massive byte-pattern diversity that 363K instruction examples cannot provide. The model achieves reasonable BPB on in-distribution text via teacher forcing (1.7-2.2 BPB on SlimOrca-format passages) but cannot generate coherently in autoregressive mode.

The BLT paper never fine-tunes on a small dataset — their closest experiment initializes from Llama 3.1 but still does continued pre-training on hundreds of billions of bytes. A viable path forward would be pre-training the encoder/decoder on a large byte corpus before fine-tuning on instruction data.

## Project Structure

* `ttblt/bltqwen.py` — all model components (tokenizer, encoder, decoder, main model, generation)
* `ttblt/entropy_model.py` — byte-level entropy model for dynamic patching
* `full_finetune_single_device.py` — training recipe
* `local_generate.py` — inference recipe
* `scripts/` — evaluation and utility scripts (entropy model training, patch precomputation, BPB evaluation)
